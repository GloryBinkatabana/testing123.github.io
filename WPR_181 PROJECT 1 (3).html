History of the Internet

Probably the first thing that comes to mind when I ask you is, "Who started the internet?" In the 1970s, Bob Kahn and Vint Cerf created the Internet. They started the process of creating what is now known as the "internet." It was the outcome of a different research project known as ARPANET, or Advanced Research Projects Agency Network. The Defence Team of the United States of America originally intended for this to be a communications network that could withstand a nuclear assault. It ultimately developed into a productive national experimental packet network. However, when did the first Internet go live? It is thought that on August 6, 1991, the public launch of the World Wide Web.

Evolution of the internet

Even though it was created much earlier, the Internet didn't become widely used in homes until the 1990s. How many firms and households began modifying their working practises and connecting their laptops and other gadgets to the Internet may be used to measure the rise of the Internet. However, it was only at this period that the idea of hypertext transfer protocol (HTTP) as we know it today was developed. This allowed users to exchange information by accessing the same web sites on several devices.

Since the internet's birth, its user base has dramatically increased. The number of interconnected computer networks has also increased tremendously as a result. Initially, there were just less than 10 computers connected. Direct connections between 440 million computers now make life easier for individuals all around the world. For individuals who have access to the Internet, sharing information and expertise has become incredibly simple. China, with 1.4 billion users, is the nation with the most internet users, followed by India with 1.3 billion and the United States with little over 0.3 billion.

Internet Protocol

Internet Protocols are a collection of guidelines that control how people communicate and share data online. To transmit data, both the transmitter and the recipient must adhere to the same protocols. Let's use a language as an example to help us better comprehend it. Any language has its own vocabulary and grammar, which we must understand to communicate. Like this, anytime we use the internet to browse a website or send data to another device, such actions are subject to a set of guidelines known as internet protocols.

Data is arranged into tiny units called packets, which are how the internet and many other data networks operate. The underlying technology and software split up each large data packet transferred between two network devices into smaller packets. According to the protocols the network supports, each network protocol specifies the requirements for how its data packets must be organised in particular ways.

Types of internet protocols include:

  • TCP/IP(Transmission Control Protocol/ Internet Protocol)
  • SMTP (Simple Mail Transfer Protocol)
  • PPP (Point-to-Point Protocol)
  • FTP (File Transfer Protocol)
  • SFTP (Secure File Transfer Protocol)
  • HTTP (Hyper Text Transfer Protocol)
  • HTTPS (Hypertext Transfer Protocol Secure)
  • TELNET (Terminal Network)
  • POP3(Post Office Protocol 3)
  • IPv4 (Internet Protocol Version 4),
  • IPv6 (Internet Protocol Version 6)
  • ICMP (Internet Control Message Protocol)
  • UDP (User Datagram Protocol)
  • IMAP(Internet Message Access Protocol)
  • SSH (Secure Shell) Gropher

Web Application Architecture

Overview

A web app architecture shows the arrangement of all the software elements, including middleware, databases, and apps, as well as how they work together. It defines how the data is sent through HTTP and makes sure that the client-side server and the backend server can understand. Furthermore, it guarantees that all user requests contain correct data. It provides authentication and permission-based access while managing and creating records. The dependability, interoperability, and future IT requirements of your business are determined by the design you choose. Therefore, it is crucial to comprehend the elements that make up the architecture of web programmes.

Web Application Architecture Components

Web Browser

The main element that engages with users, collects input, controls display logic, and regulates user interactions with the programme is the browser, also known as the client-side component or the front-end component. When necessary, user inputs are also checked.

Web Server

The web server, sometimes referred to as the backend component or the server-side component, manages the whole application's operations while handling the business logic and processing user requests by sending them to the appropriate component. It can manage and managing requests from a range of clients.

Database Server

The data needed by the programme is provided by the database server. It manages tasks involving data. Database servers may handle business logic with the use of stored procedures in a multi-tiered design.

What is 3-tier architecture

The user interface on the client side and the backend system, which is often a database server, make up a standard two-tier design. In this case, the database server or user interface include the business logic. The drawback of 2-tier design is that performance suffers as user density rises. Additionally, there are additional security issues raised by the database's direct communication with the user device. Systems for managing content and reservations for trains are two examples of applications that are frequently created utilising this architecture.

A three-tier architecture consists of the following layers:

  1. Application layer/Business layer
  2. Presentation layer/client layer
  3. Data layer

Client side and Server side engineering

Client-side software operates on a user's web browser, whereas server-side programmes run on a server. In contrast to server-side web development, which manages data behind the scenes, client-side web development involves interaction and data presentation.

Web development is an umbrella phrase for many different types of website construction, not just one specific job. Client-side, server-side, and full-stack development are the three main types of web development. For a website to work, client-side and server-side programmes are required. To build a product that is well designed and works properly, skilled developers who work in different sectors frequently need to collaborate closely.

On the other hand, full-stack engineers possess both client- and server-side web development expertise. Full-stack developers are frequently employed by businesses to maintain and manage their online presence. We'll discuss server-side vs. client-side web development in this post. We'll also go through how to pick up the programming languages you'll need to get a job in the field.

Client-side

All elements a site visitor sees and interacts with are created by client-side developers. Client-side developers employ their coding prowess to produce dynamic websites and online apps that are aesthetically pleasing, useful, and practical. Every element of a website that consumers view or interact with is created by these programmers. Virtually any visible element that appears in a web browser or requires user interaction falls under this discipline, including homepages, shopping sites, slideshows, and more.

On the client's device, all this software is active. Since user interaction occurs on the "front" of a web page, client-side developers are often referred to as "front end" programmers. These developers are well paid for their job. Of course, pay may differ depending on the business and the area. JavaScript, HTML, and CSS are three popular scripting languages that front end developers use to write client-side code.

Server-side

Data communication between websites and servers is coded by server-side developers.

We must first discuss how a website functions and how it differs from an image or an interactive text piece to comprehend what a server-side developer performs. A website's client-side functions as little more than a platform for information presentation and gathering. To deliver the services, you want, the website interacts with web servers, utilising them to retrieve and transfer data from databases.

The server-side code that enables this data transmission is designed, created, and maintained by these software engineers. Backend developers are another name for these programmers. They operate in the background, ensuring that everything functions properly on the application servers. These developers are often fluent in server-side programming languages including Python, SQL, Ruby on Rails, PHP, and Java.
